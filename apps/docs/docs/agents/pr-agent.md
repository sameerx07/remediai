# PR Agent

The PR Agent creates a branch, applies the approved fix recommendation as a code patch, and opens a draft pull request in Azure DevOps Repos. It is a Phase 2 agent and **never runs without explicit human approval**.

---

## Responsibility

- Create a new branch from the default branch of the target repository.
- Apply the `suggested_change` from the approved `Recommendation` as a patch.
- Commit the change with a standard RemediAI commit message.
- Create a **draft** pull request tagged for human review.
- Record the branch name and PR URL in the incident state.

---

## Trigger

The PR Agent runs only after a human explicitly approves a recommendation via the dashboard. The approval event is:

1. Stored in the database with the approver identity and timestamp.
2. Published to the Service Bus `ApprovalEvent` topic.
3. Consumed by the Agent Worker, which starts the Phase 2 pipeline.

**There is no timer, automatic trigger, or scheduled run.** The PR Agent will not execute without a database-recorded approval event for the incident.

---

## Input fields

| Field | Source |
|-------|--------|
| `recommendations` | Approved `Recommendation` (single item, rank specified by user) |
| `incident_id` | For branch naming |
| All `root_cause_json` + `code_snippets` | For PR description |

---

## Output fields

| Field | Type | Description |
|-------|------|-------------|
| `pr_branch` | `str` | Branch name created |
| `pr_url` | `str` | URL of the draft PR |

---

## Branch naming

```
remedia/{incident_id[:8]}/{recommendation_rank}
```

Example: `remedia/3fa85f64/1`

---

## Commit message

```
[RemediAI] Fix for incident {incident_id[:8]}: {recommendation.title}

Root cause: {root_cause_json.likely_cause}
Confidence: {recommendation.confidence:.0%}

Auto-generated by RemediAI. Human review required before merge.
Incident: {dashboard_url}/incidents/{incident_id}
```

---

## PR creation rules

- PR is always created as a **draft**.
- `autoComplete` is **never** set.
- PR title: `[RemediAI] {recommendation.title}`
- PR description includes: root cause summary, recommendation, code diff context, link to incident, and the [Validation Agent](./validation) report.
- Reviewers are set to the configured `AZURE_DEVOPS_REVIEWER_GROUP` (default: the ADO project default reviewers).

---

## What the PR agent does NOT do

- Execute code from LLM outputs.
- Run tests on the patched branch.
- Set auto-complete.
- Merge to any branch.
- Create more than one PR per approved recommendation.

---

## Example PR description

```markdown
## RemediAI — Automated Fix Proposal

**Incident:** [3fa85f64](https://dashboard.example.com/incidents/3fa85f64)  
**Exception:** `System.NullReferenceException` in `UserService.GetById`  
**Priority:** High  

### Root cause

A NullReferenceException was thrown in UserService.GetById at line 42 because the database query returned null and the result was dereferenced without a null check.

### Applied recommendation (rank 1, confidence 93%)

Add null guard before dereferencing user object.

```csharp
var user = await _repository.GetByIdAsync(id);
if (user is null) throw new NotFoundException($"User {id} not found");
```

### Files changed

- `/src/Services/UserService.cs` (2 lines)

---

> ⚠️ **This PR was generated by AI.** Review all changes carefully before merging.  
> RemediAI never auto-merges. Human approval is required.

[RemediAI Validation Report — ✅ Passed]
```

## Configuration

| Env var | Description |
|---------|-------------|
| `AZURE_DEVOPS_DEFAULT_BRANCH` | Branch to create the fix branch from (default: `main`) |
| `AZURE_DEVOPS_REVIEWER_GROUP` | ADO group to add as reviewers |
| `DASHBOARD_BASE_URL` | Dashboard URL for incident links in PR descriptions |
